# 面向对象三大特性

面向对象编程的设计思想是以对象为中心，**通过对象之间的交互来完成功能**，灵活性高、可扩展性强

而面向过程是将问题拆解为函数调用流程，性能高

## 一、三大特性

### 1. 继承

**概念**：继承可以使得子类具有父类的属性和方法，并且可以重新定义、追加属性和方法

**好处**：提高了代码的复用性

### 2. 封装

**概念**：隐藏对象的属性和实现细节，仅对外公开接口

**好处**：

1、保障了安全，因为限制了对私有成员的直接访问

2、可以在不改变接口的情况下修改接口的实现

### 3. 多态

**概念**：一个父类的引用可以指向不同类型的子类实例，从而调用不同的方法

**好处**：调用时不用在意对象的实际类型，简化代码

**实现方法**：重载与重写

## 二、重载与重写

### 1. 重载

**同一个类**中，可以根据传参的不同，来从多个同名方法中选择具体的方法去执行，发生在编译期

### 2. 重写

在继承关系中，**子类**重新定义一个与父类方法签名完全相同的方法，发生在运行期

**什么时候不能重写**：

父类中的函数被以下关键字修饰

1、private：因为子类不可见

2、static

3、final

### 3. 对比

| 区别点     | 重载     | 重写                                                         |
| ---------- | -------- | ------------------------------------------------------------ |
| 发生范围   | 同一个类 | 子类                                                         |
| 参数列表   | 必须修改 | 一定不能修改                                                 |
| 返回类型   | 可修改   | 子类方法返回值类型应比父类方法返回值类型更小或相等           |
| 异常       | 可修改   | 子类方法声明抛出的异常类应比父类方法声明抛出的异常类更小或相等 |
| 访问修饰符 | 可修改   | 一定不能做更严格的限制（可以降低限制）                       |
| 发生阶段   | 编译期   | 运行期                                                       |
