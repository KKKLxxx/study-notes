# 跨域、CSRF、SQL注入、XSS

## 一、跨域

跨域问题指的是，浏览器的同源策略规定：请求方与被请求方的协议、域名（相同顶级域名也不行）、端口三者必须完全相同，才能够正常获取请求的结果（可以发送请求，但无法读取）。同源策略避免了恶意网站盗用用户的cookie从而导致用户信息泄露（浏览器自携带cookie，即使攻击者不知道cookie，也能够使用cookie），但也对正常的前后端访问造成了限制

解决跨域问题通常就是要在服务端的响应头中设置一个字段，表示允许的访问源（CORS方案）

## 二、CSRF跨站请求伪造（Cross Site Request Forgery）

CSRF的原理就是恶意网站盗用用户的cookie从而执行恶意操作，虽然无法获取请求结果，但是可以发送请求

解决方式就是使用CSRF token，这个token是由服务端生成的，由于同源策略，其他网站无法获取这个token，所以保证了安全

## 三、SQL注入

SQL 注入的原理是，用户输入被当成了 SQL 代码的一部分执行（比如传入的password为`or 1=1`，这样就会导致条件永远为真，从而绕过登录）

解决办法就是不要拼接sql，而是要预编译sql，再填充参数

MyBatis的`#{}`就是预编译方式，所以是安全的；而`${}`是拼接，是不安全的。仍然保留`${}`是因为某些sql无法参数化，比如动态的表名称，可以使用白名单方式保证安全

## 四、XSS跨站脚本攻击（Cross Site Scripting）

xss的原理和sql注入差不多，只不过注入的内容是JS代码

解决方式就是要对特殊字符进行过滤和转义（比如`<script>`），还可以设置httponly cookie，这样js脚本就无法获取cookie
